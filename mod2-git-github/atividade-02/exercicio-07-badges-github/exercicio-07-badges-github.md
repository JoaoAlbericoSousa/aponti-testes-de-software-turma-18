# 🏷️ Badges do GitHub: O Cartão de Visita do seu Projeto

As **badges** (ou escudos) são ferramentas visuais poderosas instaladas no topo do ficheiro `README.md`. Elas servem para comunicar instantaneamente o estado atual do seu projeto a recrutadores, contribuidores e utilizadores.

---

## ⚡ Inserção de Escudos Dinâmicos

Os escudos dinâmicos mudam automaticamente de cor ou valor assim que o código do projeto é alterado. Eles ligam-se a ferramentas externas para exibir métricas em tempo real, tais como:

* **Testes Automatizados:** Indica se a última versão passou nos testes.
* **Qualidade de Código:** Mostra a percentagem de cobertura de testes.
* **Manutenção:** Exibe a versão atual do software ou o número de falhas abertas.

---

## 🛠️ Exemplos Práticos

Abaixo estão alguns exemplos de badges dinâmicas e estáticas geradas através do serviço [Shields.io](https://shields.io).

### 🟢 Status de Integração Contínua (CI)
Indica se os testes automatizados do GitHub Actions passaram com sucesso.
```markdown
![GitHub Actions Status](https://shields.io)
```
*Resultado visual:* ![Build Passing](https://shields.io)

### 📦 Última Versão do Projeto (Release)
Procura automaticamente a versão mais recente lançada no GitHub.
```markdown
![GitHub release](https://shields.io)
```
*Resultado visual:* ![Versão v1.2.3](https://shields.io)

### 🐛 Contagem de Problemas (Issues)
Mostra o número exato de bugs ou tarefas pendentes em tempo real.
```markdown
![GitHub issues](https://shields.io)
```
*Resultado visual:* ![Issues 5](https://shields.io)
